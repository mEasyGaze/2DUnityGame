# 遊戲企劃書

## 1. 遊戲概述 (Game Overview)

*   **遊戲類型**：2D RPG、回合制策略 (Turn-Based Strategy)、戰術模擬
*   **遊戲平台**：PC (Windows/Mac)
*   **核心體驗**：
    *   **預判與規劃**：非即時執行的戰鬥，玩家需預先規劃全隊行動，並透過「沙盤預演」觀察結果。
    *   **陣型與職責**：強調前衛與後衛的切換，以及體力資源的控管。
    *   **沈浸式敘事**：結合探索、對話選擇與即時運算的過場劇情。

## 2. 故事背景 (Story & Setting)

*   **世界觀**：一個劍與魔法的奇幻世界，城鎮周邊存在著威脅人類的「魔獸」。人類組成了「魔獸團」進行防禦與狩獵。
*   **序章劇情**：
    *   主角的父親前往魔獸團工作，卻粗心地忘記帶武器（爸爸的劍）。
    *   主角在家中發現媽媽留下的紙條，決定承擔起送劍的任務。
    *   在路途中遭遇魔獸襲擊（初次戰鬥教學），並被神秘強者所救，以此為契機踏上冒險旅程。
*   **主要角色**：
    *   **主角**：新手冒險者，從跑腿任務開始成長。
    *   **魔獸團衛兵**：引導主角的 NPC，提供基礎任務。
    *   **藥草商人**：負責物資補給，隨任務進度解鎖新商品。
    *   **神秘人**：戰力強大的救命恩人，引導主線劇情。

## 3. 戰鬥系統 (Combat System) - 核心玩法

本遊戲採用 **「階段式指令戰鬥」** 系統，強調戰術規劃而非反應速度。

### 3.1 戰場結構
*   **站位 (Grid Position)**：雙方各 4 個位置，共 8 格。
    *   **前衛 (Vanguard)**：承擔傷害，主要物理輸出。
    *   **遠程 (Ranged 1 & 2)**：後排輸出，較少受到攻擊。
    *   **後勤 (Support)**：負責指揮官技能，通常不直接參與普通交戰。

### 3.2 戰鬥流程 (Turn Flow)
1.  **玩家規劃階段 (Player Planning Phase)**：
    *   **引導式操作**：系統依序引導玩家為「前衛」與「遠程」單位下達指令（共 4 個行動階段）。
    *   **戰術預演**：玩家下達指令時，系統會即時模擬（扣除體力、移動模型），讓玩家預覽結果。
    *   **容錯機制**：提供「取消單步」、「重置所有規劃」、「返回」功能。
2.  **敵人規劃階段 (Enemy AI Phase)**：
    *   敵方 AI 根據性格 (Personality) 與戰術參謀 (Scorers) 進行決策。
3.  **行動執行階段 (Execution Phase)**：
    *   依照順序（玩家前衛 -> 敵人前衛 -> 玩家遠程 -> 敵人遠程）結算所有行動。
    *   此時才會造成實際傷害與數值變更。

### 3.3 資源與行動
*   **狀態指標**：
    *   **HP (紅)**：歸零陣亡。
    *   **體力 (黃)**：行動的能源。攻擊(-3)、交換(-2)、技能(視設定)。
    *   **護盾 (藍)**：優先抵擋傷害。
*   **指令集**：
    *   **攻擊**：基礎傷害。
    *   **防禦**：減傷，消耗少量體力。
    *   **休息**：回復大量體力。
    *   **交換位置**：戰術核心，前後排互換以分擔傷害或調整輸出位。
    *   **技能**：消耗體力發動特殊效果（穿透、治療、Buff）。
    *   **道具**：使用背包中的消耗品（如紅藥水）。
    *   **指揮官技能**：一場戰鬥限用一次的強力技能（全體 Buff/治療/攻擊）。

### 3.4 智能 AI
*   AI 採用 **評分系統 (Scoring System)** 而非固定的行為樹。
*   **軍師 (Personality)**：決定 AI 的風格（如：激進、保守）。
*   **參謀 (Scorers)**：評估斬殺線、自身血量、隊友狀態，為每個行動打分，AI 選擇分數最高者執行。

## 4. 探索與互動 (Exploration & Interaction)

*   **移動方式**：2D 平面移動 (Top-down)。
*   **物件互動**：
    *   **拾取**：直接獲得地面物品（如：爸爸的劍）。
    *   **採集**：需要特定工具或時間的資源點（如：生命礦石），採集後會枯竭並隨時間重生。
    *   **容器**：寶箱或木桶，可能上鎖（需鑰匙）。
    *   **機關**：開關、拉桿，用於解謎。
    *   **障礙物**：需特定道具才能移除的阻擋物（如：藤蔓）。
*   **UI 反饋**：所有採集與互動皆有「進度條 (Progress Bar)」顯示，增加沈浸感。

## 5. RPG 系統功能 (RPG Systems)

### 5.1 任務系統 (Quest)
*   **結構**：主線與支線並存。
*   **類型**：
    *   **對話類**：傳話、回報。
    *   **收集類**：取得特定數量的道具（如：5個藥草）。
    *   **討伐類**：擊敗特定敵人。
*   **特色**：
    *   支援 **前置任務** 檢查。
    *   支援 **可重複 (Repeatable)** 任務。
    *   任務狀態直接影響 NPC 對話內容（動態對話分支）。

### 5.2 背包與物品 (Inventory & Item)
*   **物品類型**：任務道具、消耗品、裝備、材料。
*   **功能**：堆疊、丟棄（會在場景生成掉落物）、交易。
*   **交易系統**：
    *   買賣介面支援拖曳操作。
    *   商店庫存可隨 **任務完成** 或 **時間經過** 動態解鎖或刷新（如：完成任務後解鎖紅藥水）。

### 5.3 隊伍管理 (Party)
*   **角色數據**：基於 ScriptableObject 的模板（基礎數值、成長率、技能池）。
*   **實例化**：每位角色擁有獨立的運行時狀態（等級、當前 HP）。
*   **編成**：玩家可自由調整戰鬥隊伍的成員與順序。

### 5.4 世界系統
*   **時間流逝**：遊戲內有天數計算，影響資源重生與商店刷新。
*   **存檔系統**：
    *   **全局存檔**：金錢、任務、背包。
    *   **場景存檔**：記錄地圖上每個物件的狀態（寶箱是否開過、礦脈是否枯竭），確保玩家體驗連貫。

## 6. 美術與介面 (UI/UX)

### 6.1 視覺風格
*   2D 像素或手繪風格（根據使用的 Assets 決定）。
*   **特效 (VFX)**：戰鬥中的傷害飄字（不同顏色區分傷害、治療、護盾）、打擊特效。

### 6.2 介面設計
*   **HUD**：
    *   頂部資訊欄：顯示當前戰鬥回合數、操作指引文字。
    *   戰鬥日誌：左下角簡易顯示（半透明），另有按鈕可開完整歷史紀錄。
*   **互動面板**：
    *   戰術規劃板：視覺化顯示 8 個行動格。
    *   技能/道具選擇：網格狀排列，滑鼠懸停顯示 Tooltip。
*   **教學引導**：
    *   條件觸發：根據血量低、初回合、初次戰鬥等條件觸發。
    *   圖文並茂：支援圖片與影片演示。

## 7. 技術架構摘要 (Technical Architecture)

基於 Unity 引擎開發，採用模組化與數據驅動 (Data-Driven) 架構。

*   **核心模式**：單例模式 (Singleton) 管理各系統 (GameManager, BattleManager, InventoryManager)。
*   **數據儲存**：ScriptableObject (物品、技能、敵人、劇情、教學) + JSON (存檔)。
*   **解耦設計**：
    *   **事件中心**：`GameEventManager` 處理跨系統溝通。
    *   **輸入系統**：`InputSystem` 與邏輯分離，支援按鍵自定義。
*   **戰鬥核心**：
    *   **狀態快照 (Snapshot)**：將數據與表現分離，實現無副作用的戰術預演。
    *   **指令模式 (Command Pattern)**：所有行動封裝為 `ActionPlan`。
